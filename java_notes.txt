## Java
multi-platform, object-oritneted programming language
functions are compiled into classes
Java Virtual Machine loads classes then gets compiled to bytecode
JVM is comparatively slower

## class
classes have members
methods are members that do things
properties are members that store things
public/private - access modifier
static - states whether instance of class is needed

## variable

    ex. Integer a;

## array
# Type name[] = new Type[number of items]
    ex. Integer intArray[] = new Integer[1];

## generics
# generic is a way to dynamically set the type for a parameter
    ex. class Test<t> {
            T obj;
            Test(T obj) {
                this.obj = obj;
            }
            public T getObject() {
                return this.obj;
            }
        }
        class Main {
            public static void main(String[] args) {
                Test<Integer> intObj = new Test<Integer>(12);
                Test<String> strObj = new Test<String>("hello");
            }
        }

find nuances:
primitive types
    are grouped in arrays
    can be arrayed when equaled to elements in {}
non-primitive types
    are grouped in lists
    lists cannot contain primitive types

length/length()/size()
    length is for array
    .length() is for String
    .size() is for list

==/equals()
    == is for reference comparison
    .equals() is for content comparison

int/Integer
    int is primitive
    Integer is non-primitive
